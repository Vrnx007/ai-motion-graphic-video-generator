# Internal benchmark rubric (20-video sprint)

Use this when generating showcase videos for real or permissioned sites. Score each 1–5; aim for 4+ before positioning as “premium.”

## Sites to run (suggested)

1. Stripe marketing (structure / density reference)
2. Linear
3. Notion
4. Vercel
5. Calendly
6. Slack enterprise landing
7. A mid-stage B2B SaaS you have permission to film
8–20. Rotate your ICP list (devtools, fintech, HR tech, etc.)

## Criteria

| # | Criterion | What “pass” looks like |
|---|-----------|-------------------------|
| 1 | Motion depth | Not only zoom/fade: stagger, parallax, masked reveal, or continuous micro-motion in most scenes |
| 2 | Typography | Short punchy lines; readable sizes; hierarchy between headline / sub / body |
| 3 | Brand fidelity | Colors and logo feel intentional; no random palette drift |
| 4 | Asset use | Curated images appear framed (device / card), not stretched edge-to-edge |
| 5 | Story arc | Clear hook → problem/context → product → proof → CTA ordering |
| 6 | Template variety | No three identical templates in a row; testimonial/integration scenes when data exists |
| 7 | Audio bed | Music level sensible; no jarring jumps if SFX appear in legacy JSX path |
| 8 | Duration honesty | Total time matches slider; no long dead air at end |
| 9 | Export | WebM export completes; share page loads with proxy images when using share token |
| 10 | Failure modes | If one scene fails validation, stitcher fallback does not break whole timeline |

## Workflow

1. Run extract → curate images → generate script → generate all scenes → preview.
2. Note failing scene index, template name, and prompt snippet.
3. File issue under `docs/internal/benchmark-notes.md` (create if needed) or your tracker.

## Target

Average score ≥ 4.0 across first 10 runs before heavy outbound sales.
